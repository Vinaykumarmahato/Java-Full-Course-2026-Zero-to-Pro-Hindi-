# 🧠 Topic: Java Type Casting (Implicit & Explicit)

## 🚀 Introduction
- **Type Casting क्या होता है?** जब हम एक डेटा टाइप (Data Type) की वैल्यू को दूसरे डेटा टाइप में असाइन करते (डालते) हैं, तो उसे **Type Casting** कहा जाता है।
- 💡 **ज़बरदस्त Analogy:** इमेजिन करें आपके पास पानी के 2 बोतल्स हैं: एक 5 लीटर का और एक 1 लीटर का।
  - 1️⃣ **(Valid)**: अगर 1 लीटर वाले बोतल का पानी आप 5 लीटर वाले बोतल में डालेंगे, तो वो आसानी से फिट हो जाएगा (No data loss)।
  - 2️⃣ **(Invalid/Risky)**: अगर 5 लीटर वाले बोतल का पानी आप 1 लीटर वाले बोतल में ज़बरदस्ती डालेंगे, तो पानी गिर जाएगा / बर्बाद हो जाएगा (Data loss)।
  - जावा में मेमोरी साइज़ के साथ बिल्कुल यही कॉन्सेप्ट लागू होता है! 

---

## 🔁 1. Implicit Type Casting (ऑटोमैटिक)
जब हम **छोटे साइज़** के डेटा टाइप को **बड़े साइज़** के डेटा टाइप में डालते हैं। यह जावा जेवीएम (JVM) द्वारा **अपने आप (Automatically)** हो जाता है और इसमें कोई डेटा लॉस नहीं होता।
- **Example:** `byte` (1 Byte) को `int` (4 Bytes) में डालना।
```java
byte smallBox = 10;
int bigBox = smallBox; // ✅ Auto magical! 
```

## ⚠️ 2. Explicit Type Casting (ज़बरदस्ती)
जब हम **बड़े साइज़** के डेटा टाइप को **छोटे साइज़** के डेटा टाइप में डालते हैं। जावा इसके लिए डायरेक्ट परमिशन नहीं देता (एरर देता है)। हमें जेवीएम (JVM) को **ज़बरदस्ती (Forcefully)** बताना पड़ता है।
- **खतरा:** इसमें डेटा लॉस (Data Loss) होने के चांसेस होते हैं!
- **Example:** `double` (8 Bytes) को `int` (4 Bytes) में डालना।
```java
int bigBox = 250;
// byte smallBox = bigBox; ❌ (Error)
byte smallBox = (byte) bigBox; // ✅ (Forceful casting)
```

## ✂️ 3. Truncation (Rounding Towards Zero)
टाइप कास्टिंग इस्तेमाल करते समय डेटा लॉस कैसे होता है? इसके लिए ये 2 नियम याद रखें:
1. **Decimal to Integer:** जब हम `float` / `double` को `int` में कास्ट करते हैं, तो पॉइंट के बाद का सारा डेटा गायब (कट) हो जाता है। 
   - *Example:* `5.6` को `int` में कास्ट करने पर आउटपुट **`5`** आएगा (Rounding towards zero)।
2. **Integer Division:** जावा में अगर दो `int` नंबर्स को डिवाइड किया जाए, तो जेवीएम आउटपुट भी `int` ही देता है (भले ही हम उसे `float` वेरिएबल में स्टोर क्यों न करें)।
   - *Example:* `12 / 5` का आउटपुट `2.0` आएगा, न कि `2.4`! (पॉइंट `0.4` लॉस हो जाएगा)।

## 🔄 4. Module Operations (Range Wraparound)
अगर हम किसी बड़े नम्बर को छोटे डेटा टाइप (जैसे `byte`) में फोर्स करके डालें, जिसकी रेंज सिर्फ `-128` से `127` है, तो क्या प्रिंट होगा?
- **Formula:** `Assigned Value % Total Possible Values (256)`
- *Example:* `(byte) 256` कास्ट करने पर रिमाइंडर आएगा `0`, इसलिए आउटपुट `0` होगा। 
- *Example:* `(byte) 576` कास्ट करने पर रिमाइंडर आएगा `64`, इसलिए आउटपुट `64` होगा।

## 🚀 5. Type Promotion in Expressions
जब हम कैलकुलेशन (Arithmetic Operation) करते हैं— जैसे मल्टीप्लाई या ऐडिशन— तो इंटरमीडिएट वैल्यू डेटा टाइप के रेंज से बाहर जा सकती है। 
- इसलिए कैलकुलेशन करते समय जावा ऑटोमैटिकली छोटे डेटा टाइप्स (`byte`, `short`, `char`) को **`int`** में 'प्रमोट (Promote)' कर देता है।
- *Example:* `byte a = 30; byte b = 40;` इन दोनों का मल्टीप्लिकेशन `1200` है जो कि बाइट की रेंज से बाहर है, लेकिन फिर भी हम इसे `int c = a * b;` लिखकर आसानी से स्टोर कर सकते हैं क्योंकि जेवीएम इंटरनली इसे इंट में 'प्रमोट' कर चुका है।

**Type Promotion Flow:**
`byte` ➔ `short` ➔ `char` ➔ `int` ➔ `long` ➔ `float` ➔ `double`
