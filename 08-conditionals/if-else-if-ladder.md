# 🪜 The `if-else-if` Ladder

## 🤔 When to use what? (The Rule of Thumb)
अभी तक हमने दो कंडीशनल स्टेटमेंट्स पढ़े हैं। आपको कब कौन सा यूज़ करना है, इसका एक बहुत ही सिंपल और गोल्डन रूल है:

1. **सिर्फ़ 1 Condition है?** ➡️ Use `if` 
   - *(e.g: सिर्फ चेक करना है कि राम के पास ₹10 हैं या नहीं)*
2. **Exactly 2 Conditions हैं?** ➡️ Use `if-else` 
   - *(e.g: Number Even है या Odd, Vote दे सकते हो या नहीं)*
3. **2 से ज़्यादा (Multiple) Conditions हैं?** ➡️ Use `if-else-if` Ladder
   - *(e.g: Grading System - Grade A, Grade B, Grade C, या Fail)*

इस तीसरे वाले कांसेप्ट को हम **`if-else-if ladder`** भी बोलते हैं। नाम सुनकर कंफ्यूज़ मत होना!

---

## 🛠️ Syntax 

```java
if (condition1) {
    // अगर Condition 1 सही है, तो यह चलेगा
} 
else if (condition2) {
    // अगर Condition 1 गलत है, लेकिन Condition 2 सही है, तो यह चलेगा
} 
else if (condition3) {
    // अगर ऊपर के दोनों गलत हैं, लेकिन Condition 3 सही है, तो यह चलेगा
} 
else {
    // अगर ऊपर की सारी Conditions गलत हो गयीं, तो आख़िर में यह वाला कोड चलेगा
}
```

---

## 🏃 Execution Flow (यह कैसे काम करता है?)
इसे आप दरवाज़ों (Doors) के गुच्छे की तरह सोच सकते हैं। प्रोग्राम एक-एक करके हर दरवाज़े (Condition) पर जा कर चेक करता है:
1. क्या कंडीशन 1 सही है? अगर **हाँ**, तो कोड रन कर दो और बाकि सारे दरवाज़ों को इग्नोर कर के सीधा बाहर निकल जाओ!
2. अगर कंडीशन 1 **गलत** है? तो दूसरे दरवाज़े (`else if`) पर जाकर चेक करो।
3. अगर बीच में कोई भी कंडीशन सही मिल जाती है, तो प्रोग्राम तुरंत वहीँ रुक कर कोड रन कर देता है। उसे आगे के कोड को चेक करने की ज़रूरत नहीं पड़ती।
4. अगर सारे दरवाज़े बंद मिले (यानी सारी Conditions False), तो डिफ़ॉल्ट रूप से `else` ब्लॉक रन हो जाता है।

## 🎯 Practical Need: Grading System
मान लीजिये हमें एक स्टूडेंट के मार्क्स के आधार पर उसका Grade पता करना है:
- **Marks >= 90**: Grade A
- **Marks >= 75**: Grade B
- **Marks >= 60**: Grade C
- **Marks < 60**: Fail

यहाँ 2 से ज़्यादा कंडीशंस हैं, इसीलिए हम **`if-else-if`** का इस्तेमाल करेंगे। (इसका Java Code आपको साथ वाली `.java` फ़ाइल में मिलेगा)।
