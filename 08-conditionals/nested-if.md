# 🪆 Nested `if` Statements in Java

## 🤔 What is a Nested `if`?
90% बिगिनर्स को नेस्टेड इफ समझने में प्रॉब्लम होती है, लेकिन यह बहुत ही सिंपल कांसेप्ट है।
सिंपल भाषा में: **"अगर किसी चीज़ के अंदर वैसी ही सेम चीज़ हो, तो हम उसे Nested बोलते हैं।"**

*नॉर्मल लाइफ के एग्ज़ांपल्स (Analogies):*
- घर के अंदर एक और घर = Nested Home
- कॉपी के अंदर एक और कॉपी = Nested Copy
- बॉक्स के अंदर एक और बॉक्स = Nested Box

बिल्कुल इसी तरह:
**"एक `if` ब्लॉक के अंदर अगर दूसरा `if` ब्लॉक लिखा हो, तो उसे Nested `if` कहते हैं!"** (If inside If).

---

## 🛠️ Syntax of Nested `if`

```java
if (outerCondition) {
    // अगर Outer Condition सही है, तब हम इसके अंदर घुसेंगे
    
    if (innerCondition) {
        // अगर Inner Condition भी सही है, तो यह कोड रन होगा
    } else {
        // अगर Inner गलत है, तो यह कोड रन होगा
    }
    
} else {
    // अगर सबसे शुरुआत वाला Outer Condition ही गलत हो गया,
    // तो अंदर का कोई भी 'if/else' चेक नहीं होगा, डायरेक्ट यह कोड रन होगा!
}
```

## 🌤️ Hands-On Project: Weather Decision App
चलिए इसे एक असली "Weather Decision App" बनाकर समझते हैं जहाँ हम **Temperature (तापमान)** और **Humidity (नमी)** के आधार पर मौसम का हाल बताएंगे।

### The Logic Breakdown:
1. **Outer Condition:** सबसे पहले हम चेक करेंगे कि क्या *Temperature 30 से ज़्यादा है?*
   - ❌ **False (30 या उससे कम है):** सीधा आउटर इल्स में जाओ और प्रिंट करो `"नॉर्मल मौसम"`। 
   - ✅ **True (30 से ज़्यादा है):** मतलब गर्मी ज़्यादा है। अब हम *अंदर जाकर* Humidity चेक करेंगे (Inner Condition)।
2. **Inner Condition:** क्या *Humidity 70 से ज़्यादा है?*
   - ✅ **True:** (गर्मी भी ज़्यादा, पसीना भी ज़्यादा) ➡️ प्रिंट करो `"खराब मौसम"`।
   - ❌ **False:** (गर्मी है पर पसीना नहीं है) ➡️ प्रिंट करो `"बढ़िया मौसम"`।

### 🚨 Golden Rule to Remember:
अगर आपकी बाहर वाली शर्त (Outer `if` Condition) **False** हो जाती है, तो जावा अंदर वाले (Inner) `if` को देखने तक नहीं जाता। वह अंदर के सारे कोड को पूरी तरह इग्नोर कर देता है!
